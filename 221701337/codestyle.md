## 代码风格

- 缩进
    >- 我的缩进风格习惯用tab，有时也使用空格。
    >- 缩进时我的每行含缩进字符数不超过80个。
    >- 如果行数过多，或者有时候有些行和有些行的数据有区别，例如第一行输出一群人的姓名，第二行输出成绩的时候，我会加以注释或者若干空行加以分割。
    >- 我的缩进一般向左对齐。
- 变量命名
    >- 当变量名或函数名由多个单字连接在一起的时候，我大多数情况会选择用下划线将单字分开。有时也会使用驼峰命名法。
    >- 全局变量首字母大写，若全局变量出线多个单字则每个单字的首字母大写。临时变量一般全部小写，若出线多个单字，每个单字用下划线分隔。部分也使用驼峰命名。
    >- 函数名尽量做到见名起义，尽量使用英文，实在不行时会使用拼音或者注释。
    >- 变量名我也将尽量取有意义的名字，避免a，b，c，d，实在想不出时才有可能使用单个字符。但是对于循环体时，例如while（i），for（int i=0），在这些情况下控制循环的字符我一般会选择单个字符，且一般会选择i，j，k。
    >- 变量或者函数的英文名过长时，我会选择缩小或者取开头3-5个字母，尽量做到不破坏原单词的意思。例如count缩写为cnt，message缩写为msg。
    >- 不同变量之间尽量做到低相似性，完全不相干的两个变量避免出现只相差一个字母的差别或者只差一个大小写的差别。
    >- 数组命名时，我也会做到见名起义，尽量与储存的数据类型挂钩，如果内容是一串字符串我一般会命名为string。
- 每行最多字符数
    >- 一般为150-160。
- 函数最大行数
    >- 80-90。
- 函数、类命名
    >- 与变量命名类似。
- 常量
    >- 字母全部大写，尽量做到见名起义。
- 空行规则
    >- 不同类之间空行。
    >- 不同方法之间空行。
- 注释规则
    >- 如果有必要注释时，一般使用//进行注释。如果要注释这个程序是干什么的话会在程序开头用/*…*/注释。
    >- 注释如果超过一个的行，我会另起一行用//注释接上上一行的内容。
- 操作符前后空格
    >- 各空一格
- 其他规则
    >- 花括号独占一行
